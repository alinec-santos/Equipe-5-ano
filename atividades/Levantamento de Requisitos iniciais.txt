Atividade 1 - Fazer o levantamento inicial: 3 histórias de usuários, 3 requisitos funcionais e 3 requisitos não-funcionais. 

-----------------------------------------------------------------------------------------------
				Histórias de Usuários 
1- Como criança usuária do jogo, quero descobrir a função de cada peça de um computador, para entender como elas se conectam e contribuem juntas para o funcionamento do sistema.
2- Como responsável por uma criança usuária do jogo, quero que meu filho(a) use um software educativo, divertido e seguro, para que ele(a) possa aprender sobre tecnologia e desenvolver interesse pela área.
3- Como educador(a), quero acompanhar o desempenho das crianças no jogo por meio de relatórios simples e visuais, para avaliar seu progresso na aprendizagem e identificar quais conceitos precisam ser reforçados.
-----------------------------------------------------------------------------------------------
				Requisitos Funcionais
1- O sistema deve permitir que o jogador monte um computador virtual interativamente, seguindo uma progressão por níveis. A montagem deve ocorrer de fora para dentro, com componentes sendo arrastados e posicionados corretamente na estrutura do computador.
2- O sistema deve oferecer ao jogador a opção de usar dicas em momentos de dificuldade, garantindo sua progressão. O botão de dica deve ter um design intuitivo e ilustrativo para facilitar o entendimento. Quando acionado, ele deve fornecer uma informação relevante e contextual para auxiliar o jogador na montagem do computador.

-----------------------------------------------------------------------------------------------
				Requisitos Não-Funcionais
1- O sistema deve explicar os assuntos de um jeito divertido e fácil de entender, com imagens e exemplos simples, pra ajudar as crianças a aprenderem mesmo que nunca tenham visto um computador por dentro.
2- O usuário deve receber feedback imediato sobre acertos ou erros na montagem dos componentes.
